# SwiftUI 代码检查

按照 MagicUI SwiftUI 开发标准规范检查代码：

1. **检查代码组织**
   - 每个 struct/class 是否在独立文件中
   - 文件名是否与类型名称一致
   - 相关组件是否组织在同一目录
   - View 扩展是否放在 `ExtView+/` 目录

2. **检查 MARK 分组**
   - 是否使用 // MARK: - 分组
   - 分组顺序是否正确：Properties → Body → Actions → Helpers → Preview
   - 是否使用 extension 隔离不同分组（可选）

3. **检查 View 扩展规范**
   - View 扩展是否返回 `some View`
   - 是否支持链式调用
   - 方法命名是否清晰（magic 前缀或介词 in、with）
   - 公共 API 是否标记为 `public`

4. **检查组件设计**
   - 是否提供合理的默认值
   - 是否支持自定义配置
   - 是否添加详细的中文注释
   - 是否包含多个 #Preview 示例

5. **检查预览代码**
   - 文件底部是否添加 #Preview
   - 是否提供多个预览变体（不同样式、尺寸）
   - 是否使用 #if DEBUG 条件编译
   - 预览是否展示实际使用场景

6. **检查跨平台兼容性**
   - 是否使用条件编译处理平台特定代码
   - 是否在两个平台（macOS 和 iOS）都能构建
   - 是否正确使用系统框架

7. **修复不规范的地方**
   - 自动修复可以自动修复的问题
   - 列出需要手动修复的问题
   - 提供修复建议

## 重要规则

- 使用中文与用户交流
- 仅执行检查和修复，无需总结
